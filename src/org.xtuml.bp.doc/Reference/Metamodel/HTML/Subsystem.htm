101. Model Class

Model Class(Obj_ID, Name, Numb, Key_Lett, Descrip, SS_ID)

Identifier *: Obj_ID

Identifier *2: Key_Lett

Identifier *3:

Description: A model class represents an abstraction of a real world thing. All instances of a class have the same characteristics and conform to the same set of rules. The characteristics of a class are captured as attributes. Each class within a domain are assigned a unique name, number, and keyletter.

Model Class.Obj_ID

Full Name: Class Identifier

Attribute Type: Base Attribute

Data Type: unique_id

Description: Uniquely identifies the class within the model.

Model Class.Name

Full Name: Class Name

Attribute Type: Base Attribute

Data Type: string

Description: Name of the class as entered in the model.

Model Class.Numb

Full Name: Class Number

Attribute Type: Base Attribute

Data Type: integer

Description: Number of the class as it appears in the model.

Data Domain: 0-9999

Model Class.Key_Lett

Full Name: Class Key Letters

Attribute Type: Base Attribute

Data Type: string

Description: Key letters of the class as they appear in the model.

Model Class.Descrip

Full Name: Class Description

Attribute Type: Base Attribute

Data Type: string

Description: This is a description of the class as entered by the user.

Model Class.SS_ID

Attribute Type: Referential Attribute

Refers to: Subsystem.SS_ID (R2)

102. Imported Class

Imported Class(IObj_ID, Obj_ID, Modl_Typ, SS_ID, Obj_Name, Obj_KL)

Identifier *: IObj_ID

Identifier *2:

Identifier *3:

Description: Classes can have interactions with classes in other subsystems. In order to capture these spanning interactions, Classes can be imported into another subsystem. Spanning interactions can be associations, event communications, or data accesses and are captured in a class diagram, communication diagram, and synchronous communication diagram, respectively. Spanning interactions provide the data for derivation of the subsystem association model, subsystem communication model, and subsystem access model.

Imported Class.IObj_ID

Full Name: Imported Class Identifier

Attribute Type: Base Attribute

Data Type: unique_id

Description: Uniquely identifies an instance of an imported class.

Imported Class.Obj_ID

Attribute Type: Referential Attribute

Refers to: Model Class.Obj_ID (R101)

Imported Class.Modl_Typ

Full Name: Model Type

Attribute Type: Base Attribute

Data Type: integer

Description: Indicates what type of model the Imported Class is in.

Data Domain: 5 indicates Class Diagram

Imported Class.SS_ID

Attribute Type: Referential Attribute

Refers to: Subsystem.SS_ID (R3)

Imported Class.Obj_Name

Full Name: Class Name

Attribute Type: Derived Base Attribute

Data Type: string

Description: This attribute provides identifying information for the Class related to this Imported Class. This is used to associate the Imported Class to its related Class when importing a subsystem containing the Imported Class into a new/different domain.

Imported Class.Obj_KL

Full Name: Class Key Letter

Attribute Type: Derived Base Attribute

Data Type: string

Description: This attribute provides identifying information for the Class related to this Imported class. This is used to associate the Imported Class to its related class when importing a subsystem containing the Imported Class into a new/different domain.
