Setting the Model Compiler

Before you can use translate an xtUML project using an eclipse-based builder, you must first configure the project with the correct model compiler to use.  There are two ways to perform this step:

·         Select the model compiler to use when creating a new xtUML Project

·         Launch the “Set Model Compiler” wizard on an existing project

 

In both cases, the selection wizard will only display model compilers that you are licensed to use.  If the desired model compiler is not listed, check licensing as there may be a configuration issue.

 

Choosing the Model Compiler during New Project Creation

This is the most common way to associate a model compiler with an xtUML Project.  When you are creating a project you often know what the target language of the translated model will be.  Alternatively, you may intend the new project to be a library of elements that will be accessed via inter-project references.  In such a case, you can choose “None” on the model compiler selection page.  When you choose “None”, no model compiler or CDT builders or support files and directories are added to the new project.  The “None” option is automatically selected by the xtUML Editor for all projects.

 

Setting the Model Compiler on an Existing Project

Alternatively, you may decide after a project is already created that you would like to use a different model compiler to translate the project to a different target language.  Or you may decide that you would like to convert the existing project to or from a library-style project.

 

You can also use this tool to update from old (deprecated) model compiler configurations.  If you find that your project builders are shown as “Invalid Tool Builder” in the Builders list, run this utility to update to the correct model compiler configuration.  The tool will also automatically add the xtUML Nature to the project if it does not exist.  Thus, you could create a Java or C project, then “Set Model Compiler” to add in the xtUML editing and translating natures.

 

To run the utility, select a project, then right-click to bring up the context menu.  Next select BridgePoint Utilities > Set Model Compiler (see Figure 1) which triggers a wizard (Figure 2) that shows the model compiler selection dialog.  Once again: this dialog will only display model compilers that are correctly licensed.

You can run the “Set Model Compiler” utility as often as required to change the project model compiler.  Note, the tool does modify the project settings and will make the project “dirty” in the eyes of your revision control system.

 

 

A Note about Converting to No Model Compiler:

When you convert a project that is associated with a model compiler to “None”, only the model compiler builders are removed.  The CDT builders are left alone.  All files and folders on disk that were previously created by the model compiler association are also not removed or modified.
