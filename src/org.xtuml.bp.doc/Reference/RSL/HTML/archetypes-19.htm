Action Translation Rules

Action translation rules are the means by which actions are translated into code.

In the xtUML models, actions can be specified for several model elements: state machine states, bridges, functions, operations, and derived attributes. In terms of the metamodel, the actions (written in OAL) are stored in an attribute for each of the classes corresponding to the modeled element: SM_ACT , S_BRG , S_SYNC , O_TFR , and O_DBATTR . In all cases, the name of the attribute is Action_Semantics , and the type is string .

Actions are translated using a technique very similar to that used by a compiler to produce machine code from a high level language. However, one major difference exists: the last step of the translation is done through a set of translation rule functions called fragment translation functions , so called because each function produces a fragment of code used to build up the translation of an action. A fragment translation function exists for each of the OAL statements.

These functions give complete control over the process of translating OAL into the target implementation of the system architecture.
