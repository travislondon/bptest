Structural Rules

Structural rules may be thought of as rules that produce the structure of the system based on the data and control in the xtUML models. BridgePoint UML Suite Rule Specification Language (RSL) is used to write these rules.

In a C++ system architecture, the structure of the code includes classes that define the data in the system, the events generated in the xtUML models, and state event matrix structures used by the event dispatching processes. Structural rules emit structure as opposed to action translation rules which are solely used to emit code for the OAL statements contained with the xtUML model actions.

Structural rules set up the structure of the code that the action translation rules use. For example, all of the events are known, their signatures, etc., in the xtUML database. The methods for generating events are specified by the structural rules. The translated actions refer to them.

Likewise structural rules generate member functions for data accesses (reads and writes) of attribute data. The action translation rules translate the OAL that reads and writes attributes into calls to the member functions emitted by the structural rules.
